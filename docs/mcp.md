# Подключение MCP-серверов к чату редактора

Редактор умеет работать с внешними **MCP-серверами** (Model Context Protocol) —
это отдельные программы, которые дают ИИ-ассистенту дополнительные «инструменты»
(поиск, чтение документов и т.п.). Поддержка включена пакетом `@theia/ai-mcp`.

Ниже — как подключить наш поисковый бэкенд **`bs-search`** (поиск по священным
писаниям) прямо в чат.

---

## 1. Куда вписывается конфигурация

Список серверов хранится в **настройках** редактора под ключом:

```
ai-features.mcp.mcpServers
```

Открыть настройки: `Файл → Preferences → Settings` (или палитра команд →
`Preferences: Open Settings (JSON)`), затем в поиске набрать `mcp`. Проще всего
редактировать JSON напрямую — нажать **Edit in settings.json**.

> Подсказки в интерфейсе (заголовок «MCP Server Configuration», команда
> «Add local MCP config...», кнопки «Start Server» / «Stop Server») идут из ядра
> Theia и остаются на английском — мы их не переводим. Русской локали для этих
> строк в стандартном пакете ядра нет.

---

## 2. Готовый пример для `bs-search`

Вставьте этот блок в `settings.json`:

```json
{
  "ai-features.mcp.mcpServers": {
    "bs-search": {
      "command": "bash",
      "args": [
        "-lc",
        "cd /Users/vedmalex/work/bs-search && exec bun run mcp:start"
      ],
      "autostart": true
    }
  }
}
```

Что здесь происходит:

- `bs-search` — имя сервера (под ним инструменты появятся в чате).
- `command` / `args` — как запускать. Схема настройки поддерживает только
  `command`, `args`, `env` — **поля `cwd` нет**. Поэтому рабочую папку
  задаём обходным путём: через обёртку `bash -lc "cd … && exec …"`.
  - `cd /Users/vedmalex/work/bs-search` — сервер `bs-search` читает базу по
    относительным путям, поэтому его нужно запускать именно из своей папки.
  - `bash -lc` (login-shell) подтягивает ваш `PATH` из профиля, чтобы нашлись
    `bun` и `node`/`npx`.
  - `exec` — чтобы сигнал остановки дошёл до самого сервера, а не до обёртки.
  - `bun run mcp:start` — это «священная» команда запуска из `bs-search`
    (внутри она выполняет `npx tsx src/mcp-server/index.ts`).
- `autostart: true` — сервер сам поднимется при следующем запуске редактора.

Если нужны переменные окружения (например, ключ для RAG-инструментов) — добавьте
блок `env` (обычный поиск ключ не требует):

```json
      "env": { "OPENROUTER_API_KEY": "..." }
```

Много инструментов и хотите экономить контекст модели? Добавьте
`"deferLoading": true` — тогда инструменты не грузятся в промпт заранее, а
модель находит их по мере надобности.

---

## 3. Как запустить / остановить сервер

- **Первый запуск.** С `autostart: true` сервер поднимется **при следующем
  перезапуске редактора** (автозапуск срабатывает только после рестарта).
  Просто перезапустите приложение после сохранения настроек.
- **Добавить через диалог.** Есть команда `AI: Add local MCP config...`
  (палитра команд) — она открывает форму добавления локального сервера и
  записывает его в те же настройки.

> ⚠️ **Важно (текущая сборка).** Отдельная панель управления MCP-серверами с
> кнопками **Start / Stop** приходит из пакета `@theia/ai-ide`, а он в этой
> сборке **не установлен**. Поэтому вручную «нажать Start» в интерфейсе пока
> нельзя — рабочий путь: прописать сервер в настройках с `autostart: true` и
> перезапустить редактор. Если нужна кнопочная панель Start/Stop — надо
> отдельно добавить `@theia/ai-ide` (это уже отдельное решение, вне текущей
> задачи).

---

## 4. Как инструменты появляются в чате

Когда сервер запущен, его инструменты подключаются автоматически:

- **Панель возможностей (capability presets).** В панели выбора возможностей
  чата появляется **группа с именем сервера** — `bs-search` — а под ней список
  инструментов. Можно включать нужные галочками.
- **Функции.** Каждый инструмент регистрируется как функция с ID вида
  `mcp_<сервер>_<инструмент>`, например `mcp_bs-search_search`.
- **Ссылки в промптах.** В тексте промпта/агента можно сослаться на инструмент
  через `~{mcp_bs-search_search}`, либо подключить **все** инструменты сервера
  разом через фрагмент `~{mcp_bs-search_tools}`. При `deferLoading: true`
  синтаксис становится `~{?mcp_bs-search_...}` (загрузка по требованию).

Инструменты, которые отдаёт `bs-search`:

`search`, `get_document`, `db_stats`, `answer`, `sources_overview`,
`research_plan`, `research_gather`, `research_batches`, `describe_document`,
`resolve_scope`, `route_model`.

---

## 5. Если что-то не работает

- Сервер не стартует после перезапуска — проверьте, что путь
  `/Users/vedmalex/work/bs-search` верный и что в нём работает `bun run mcp:start`
  из терминала.
- Инструменты не появились в панели — сервер, скорее всего, ещё не запущен
  (см. раздел 3: с текущими зависимостями нужен `autostart: true` + рестарт).
- В упакованном Electron-приложении у фонового процесса может быть урезанный
  `PATH`; обёртка `bash -lc` как раз подгружает ваш профиль и обычно решает это.
