Если в проект книги попадает не только markdown — присланный редактором docx,
таблица в xlsx, чужая презентация в pptx, скан-архив в epub, — открывать для
этого отдельную программу не нужно. Такие файлы показываются прямо внутри
редактора: своим read-only экраном, без набора текста и без правки.

## Как это открывается

Двойной щелчок по docx, odt, rtf, xlsx, xls, ods, pptx, odp или epub-файлу в
дереве файлов сразу открывает этот экран — предпросмотр здесь главнее обычного
текстового редактора, специально выбирать его не нужно. Та же кнопка работает
и вручную: для файла, выделенного в дереве, или открытого сейчас в активной
вкладке.

::action[Предпросмотр офисного документа]{command="ai-focused-editor.office.openPreview"}

Если ничего подходящего не выделено и не открыто, редактор честно попросит
сначала выбрать файл — docx, odt, rtf, xlsx, xls, ods, pptx, odp, epub, а также
старые doc и ppt (про них — ниже).

## Что показывается — по типу файла

- **Текстовые документы** (`.docx`, `.odt`, `.rtf`) — сплошной
  отформатированный текст: заголовки, списки, выделения. В `.docx` в текст
  подтягиваются и картинки, вставленные в документ.
- **Таблицы** (`.xlsx`, `.xls`, `.ods`) — каждый лист книги отдельной вкладкой
  сверху, под ней обычная HTML-таблица.
- **Презентации** (`.pptx`, `.odp`) — каждый слайд отдельной карточкой: первая
  строка текста становится заголовком, остальные превращаются в список.
- **Электронные книги** (`.epub`) — заголовок книги, оглавление по главам и
  текст первой главы.

## Что честно нужно знать заранее

:::requires{title="Чем этот предпросмотр ограничен"}
- **Из epub показывается только первая глава.** Оглавление рядом перечисляет
  все главы книги, но по нему нельзя перейти к другой главе — это просто
  список, не ссылки. Чтобы прочитать книгу дальше первой главы, открывайте её в
  отдельной программе для чтения epub.
- **Картинки на слайдах не показываются вовсе** — ни в `.pptx`, ни в `.odp`.
  Для `.pptx` экран честно предупреждает об этом отдельной строкой, если в
  файле есть встроенные изображения; для `.odp` такого предупреждения нет, хотя
  ограничение то же самое: видны только текст и заголовки слайдов.
- **Старые `.doc` и `.ppt` не поддержаны совсем.** Двойной щелчок по такому
  файлу открывает тот же экран, но вместо содержимого — честная карточка «этот
  документ нельзя показать» с советом пересохранить файл в `.docx`/`.pptx` или
  открыть его в отдельной программе. Никакого текста или картинок оттуда
  вытащить нельзя.
- **Таблицы обрезаются** — не больше 1000 строк и 50 столбцов на лист; при
  обрезке на вкладке листа появляется пометка об этом.
- **Файл тяжелее 50 МБ вовсе не открывается предпросмотром** — редактор
  предложит открыть его в отдельной программе.
- **Слишком длинная глава epub тоже обрезается** — предпросмотр показывает не
  больше мегабайта её HTML-текста, а книгу с оглавлением длиннее 500 пунктов
  обрежет и по числу глав. Для обычной книги это не встретится, но для очень
  длинного справочника — возможно.
:::

## Открыть как текст

У предпросмотра есть отдельная кнопка «Открыть как текст» — тот же файл
открывается в обычном текстовом редакторе, минуя предпросмотр. То же самое
делает отдельная команда:

::action[Открыть офисный документ как текст]{command="ai-focused-editor.office.openAsText"}

Честно об этом стоит знать: `.docx`, `.xlsx`, `.xls`, `.ods`, `.pptx`, `.odp`,
`.odt` и `.epub` внутри — это ZIP-архивы, а не обычный текст, так что открытые
«как текст» они покажут нечитаемую кашу из байтов. По-настоящему читаемым
текстом в этом виде окажется только `.rtf` — он и есть простой текст с
управляющими последовательностями. Открывайте эту команду, только если вам
специально нужно заглянуть в сырой файл, а не почитать содержимое. Вернуться от
открытого «как текст» файла назад к предпросмотру можно правым щелчком в самом
редакторе — там же есть команда предпросмотра.

Команда доступна через палитру команд и через меню `Manuscript` →
`Knowledge`, когда подходящий файл выделен в дереве или открыт в активной
вкладке.

## Изображение как текст

Для картинок в дереве файлов есть похожая, но отдельная команда — тот же
принцип, другой обработчик:

::action[Открыть изображение как текст]{command="ai-focused-editor.image.openAsText"}

Она находится в контекстном меню дерева файлов (правый щелчок по картинке) и
открывает файл как есть, в текстовом редакторе, — обычно вместо привычного
предпросмотра-картинки. Читаемым текст получится только для SVG: это обычная
разметка. Для растровых форматов (`.png`, `.jpg` и прочих) вы так же увидите
нечитаемые байты — команда полезна, только если вам действительно нужно
заглянуть в разметку файла, а не посмотреть саму картинку.
